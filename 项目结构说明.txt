【0】一句话概要
────────────────────────────────────────────────────────────────
这是一个「客服指挥中心 BI 看板」，实际只有 *一个* 真正跑业务的
HTML 档：index.html。里面用「分页切换」的方式塞了 6 个业务模组
(总览 / 组员 / 排班 / 绩效 / 日报 / 产品)。资料来自 Google Sheets，
透过「CSV 直连」加「Google Apps Script Web App」两条路进来。

其它 demo-0X.html / concept-preview.html / prototype-*.html
都是「视觉特效概念稿」，不是业务模组，不要改错档。

【4】资料流向 —— Google Sheets 怎么进到看板
────────────────────────────────────────────────────────────────

  有 *两条* 路径并存：

  路径 A：CSV 直连 (不需要 Apps Script)
  ─────────────────────────────────────────────────────
  前端直接 fetch Google Sheet 的 CSV 汇出网址：

    SYS-01 每日回复量    sheet 1lKjyN...dOLIFn1J0syvCLdjWzk  gid=1762805145
    SYS-02 班别回复量    (同上)
    SYS-03 阶段回复量    (同上)
    SYS-04 营运数据      sheet 19KeuX9iq7U-ox4IFOIUEVhNMm-kgSJhBNWtsC9h9g40  gid=860680192
    KPI 总同步           同时拉 SHEET_01 + SHEET_04 聚合

  好处：不用部署；坏处：只能「读」，不能写。

  路径 B：Google Apps Script Web App (读 + 写)
  ─────────────────────────────────────────────────────
  index.html 内有两支 GAS Web App URL：

    SheetSync.init(URL_A, key)       →  google-apps-script.gs
      用途：users / login_logs / action_logs 云端同步
      API key：bi-dashboard-sync-2026-secure-01
      动作：getUsers / upsertUser / addLoginLog / addActionLog

    SCHEDULE_API_URL = URL_B         →  google-apps-script.gs
      用途：拿「4月假表26」这个矩阵班表
      动作：?action=getLocalSchedule&sheet=4月假表26
      内部流程：GAS 收到请求 → 呼叫 LocalParser.parse(sheet)
               → 把「日期 × 早/中/夜班 × 人名 → 班/休/事假」
                 这个二维矩阵解析成 JSON → 回传前端

  LocalParser.gs 做什么？
  ─────────────────────────────────────────────────────
  专门解析业主手写的班表 Excel 格式 (「4月假表26」这类 sheet)：
    - 自动侦测表头行 (含「早班/中班/夜班」+「值班人数」关键字)
    - 自动切分三个班次区块，提取该区块下的人名列
    - 逐日读取每人的「班/休/事假/特休/病假/迟」
    - 对应到标准状态：on_duty / day_off / leave / late
    - 产出 { daily: {1:{morning:{...}, swing:{...}}}, summary:{...} }

  资料流向图：
    ┌──────────────┐
    │ Google Sheet │  (业主手写班表 / 回复量统计)
    └──────┬───────┘
           │
      ┌────┴────┐
      │         │
   CSV 汇出   GAS Web App
      │         │
      │      ┌──┴──────────────┐
      │      │ google-apps-    │
      │      │ script.gs       │
      │      │  +              │
      │      │ LocalParser.gs  │
      │      └──┬──────────────┘
      │         │  (JSON)
      └────┬────┘
           ▼
    ┌─────────────┐
    │ index.html  │  →  六个分页的图表 / KPI / 跑马灯
    └─────────────┘


【5】给其他 AI 接手时的使用说明 (业主不写程式码，请照做)
────────────────────────────────────────────────────────────────

  A. 业主是谁、怎么沟通
     ─────────────────────────────
     - 业主不会写程式码，要用「白话」沟通、不要直接贴一串 code。
     - 业主沟通时混用繁体 / 简体中文；回覆保持中文即可。
     - 业主的指令通常是「改数字」「换颜色」「加一个按钮」这种
       视觉 / 资料层面的需求，很少涉及架构重构。

  B. 看到任何需求，请先这样判断
     ─────────────────────────────
     1. 先问自己：这是「01~06 六个模组」中的哪一个？
        → 一定是在 index.html 里面改，不要去动 demo-0X。
     2. 确认是哪一页后，用 #page-xxx 搜寻到对应区段。
     3. 再看需求是：
        (a) 改文字 / 数字 / 颜色  →  直接改 HTML / style。
        (b) 改资料来源           →  查本档「第 4 节 资料流向」。
        (c) 改班表解析逻辑       →  改 LocalParser.gs，不是前端。
        (d) 改用户 / 登入 / 日志 →  改 google-apps-script.gs。

  C. 常见误区 (前任 AI 踩过的雷)
     ─────────────────────────────
     - 误把 demo-01-rift.html 当成「模组 01 总览」去改 → 完全错档。
     - 想要「做一个新的 HTML」→ 错，直接在 index.html 加分页。
     - 改了 LocalParser.gs 但忘了它跑在 Google Apps Script 端，
       不是 npm 部署，要去 script.google.com 重新贴上并发布。
     - index.html 有 60 万 bytes 非常大，直接全档读会爆，
       请用 grep/sed 定位 + Read 工具带 offset/limit。

  D. 近期业主决策 (看 git log 就知道)
     ─────────────────────────────
     - 2026-04-20：模组 06 的 68 项产品已写死 (hardcode)，
       不再从 sheet 拉 → 不要再「补回同步」。
     - 投诉数量 / 投诉金额 已固定写死 390 / 69710，
       不再从 localStorage 编辑。
     - 跑马灯会跟着所有资料源自动同步。

  E. 修改前的自保 SOP
     ─────────────────────────────
     1. 用 git status 确认干净。
     2. 修改只在 index.html 或 *.gs 里。
     3. 改完先 grep 搜旧值，确保没有漏改。
     4. 业主会要求「我先看看」→ 要等业主看过再 commit。
     5. commit message 沿用现有风格 (feat / refactor / fix)。

  F. 给业主本人 (你) 的使用提醒
     ─────────────────────────────
     这份文件你自己也要留一份，遇到下列情况翻开用：

     1. 找新 AI 帮忙时
        → 开场白先说「请先读 GitHub 仓库根目录的 项目结构说明.txt」

     2. 自己忘了某个功能在哪里
        → 翻【3】，搜模组名称就能找到对应 DOM ID

     3. 要 AI 改东西时，照这个模板说：
        ┌────────────────────────────────────────────┐
        │ 我要改的是: 模组 XX (例: 03 出勤排班)       │
        │ 档案位置: index.html                        │
        │ DOM 定位: #page-xxx (例: #page-attendance) │
        │ 改什么: [具体描述]                          │
        │ 注意: 我不写 code, 请给我可复制的命令       │
        └────────────────────────────────────────────┘

     4. 每次改完 push 前，要求 AI 让你先预览，你确认再 push。
